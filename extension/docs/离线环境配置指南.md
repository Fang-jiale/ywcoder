# YwCoder 离线环境配置指南

## 概述

本文档指导如何在离线或私有网络环境中配置 YwCoder 扩展。

## 配置文件位置

YwCoder 使用以下配置文件路径（与之前的 Claude Code 不同）：

| 配置类型 | 文件路径 | 说明 |
|---------|---------|------|
| 全局设置 | `~/.claude/settings.json` | 用户级默认配置 |
| Profile 设置 | `~/.claude/settings.<name>.json` | 多配置方案 |
| 项目设置 | `<workspace>/.claude/settings.json` | 工作区共享配置 |
| 本地设置 | `<workspace>/.claude/settings.local.json` | 工作区本地配置（不提交到 git） |
| MCP 配置 | `~/.claude/mcp.json` | MCP 服务器配置 |
| 扩展配置 | `~/.ywcoder.json` | YwCoder 扩展专属配置 |

## 使用本地已安装的 Claude CLI

如果你已经在系统上安装了 Claude CLI，可以让 YwCoder 使用它而不是内置的 CLI。

### 配置方法

编辑 `~/.ywcoder.json`（扩展配置文件）：

```json
{
  "localClaudeCliPath": "/usr/local/bin/claude"
}
```

支持的路径格式：
- **绝对路径**: `/usr/local/bin/claude`
- **用户主目录**: `~/.local/bin/claude`
- **环境变量**: `$HOME/.local/bin/claude` 或 `${HOME}/.local/bin/claude`

### 优先级

CLI 选择的优先级顺序：
1. 用户配置的本地路径 (`localClaudeCliPath`)
2. 扩展内置的原生二进制文件
3. 扩展内置的 `cli.js`

如果配置的本地路径不存在，会自动回退到内置 CLI。

### 常见安装路径

不同安装方式的典型路径：

| 安装方式 | 典型路径 |
|---------|---------|
| npm 全局安装 | `/usr/local/bin/claude` 或 `~/.npm-global/bin/claude` |
| Homebrew (macOS) | `/opt/homebrew/bin/claude` 或 `/usr/local/bin/claude` |
| 官方安装脚本 | `~/.local/bin/claude` |
| 自定义安装 | 安装时指定的路径 |

### 查找本地 Claude CLI

如果不确定 Claude CLI 安装位置，可以使用以下命令查找：

```bash
# macOS/Linux
which claude

# 或使用 find 命令
find ~ -name "claude" -type f 2>/dev/null

# 检查是否在 PATH 中
claude --version
```

## 离线环境配置步骤

### 1. 创建配置目录

```bash
mkdir -p ~/.claude
```

### 2. 复制离线配置文件

将 `offline-settings.json` 复制到用户配置目录：

```bash
cp docs/offline-settings.json ~/.claude/settings.json
```

### 3. 配置 API 端点

编辑 `~/.claude/settings.json`，修改以下环境变量：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://your-internal-api-endpoint.com/v1",
    "ANTHROPIC_AUTH_TOKEN": "your-api-key-here"
  }
}
```

支持的配置方式（按优先级排序）：

1. **环境变量**（最高优先级）
   ```bash
   export ANTHROPIC_BASE_URL="http://your-api-endpoint.com/v1"
   export ANTHROPIC_AUTH_TOKEN="your-api-key"
   ```

2. **settings.json** 中的 `env` 字段

3. **VSCode 设置**中的环境变量配置

### 4. 权限配置说明

离线配置默认：

- **允许**: 文件读写、Git 操作、常见开发工具命令
- **拒绝**: 网络访问命令 (curl, wget, ssh 等)、Web 搜索/获取
- **询问**: 危险操作 (rm -rf, dd 等)

### 5. 网络沙盒配置

```json
{
  "sandbox": {
    "enabled": true,
    "network": {
      "allowLocalBinding": true,
      "deniedDomains": ["*"]
    }
  }
}
```

## 常见问题

### Q: 配置文件修改后没有生效？

A: 
1. 确保文件路径正确：`~/.claude/settings.json`
2. 重启 VSCode 或重新加载窗口
3. 检查 JSON 语法是否正确
4. 查看 YwCoder 输出面板中的日志

### Q: 如何验证配置是否加载？

A: 在 YwCoder 聊天中输入 `/config` 查看当前生效的配置。

### Q: 如何切换到本地已安装的 Claude CLI？

A: 编辑 `~/.ywcoder.json` 文件：

```bash
# 查找本地 Claude CLI 路径
which claude

# 编辑扩展配置文件
echo '{"localClaudeCliPath": "/usr/local/bin/claude"}' > ~/.ywcoder.json
```

重启 VSCode 后生效。可在 YwCoder 输出面板查看实际使用的 CLI 路径。

## OpenAI 格式适配器

YwCoder 内置了 OpenAI 到 Anthropic 格式的适配器，可以无缝使用 OpenAI 兼容的 API（如 LM Studio、Ollama、vLLM 等）。

### 自动检测

适配器会自动检测以下情况并启用：
- `ANTHROPIC_BASE_URL` 包含 `localhost`、`127.0.0.1`、`lmstudio`、`ollama` 等关键词
- 扩展配置中启用了 `useOpenAIAdapter`

### 手动启用

编辑 `~/.ywcoder.json`：

```json
{
  "useOpenAIAdapter": true,
  "openAIModel": "gpt-4"
}
```

### 配置示例

#### LM Studio (本地)

```bash
# ~/.claude/settings.json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:1234/v1",
    "ANTHROPIC_AUTH_TOKEN": "not-needed"
  }
}

# ~/.ywcoder.json (扩展配置)
{
  "useOpenAIAdapter": true,
  "openAIModel": "local-model"
}
```

#### Ollama (本地)

```bash
# 启动 Ollama 并确保 API 可用
ollama serve

# ~/.claude/settings.json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:11434/v1",
    "ANTHROPIC_AUTH_TOKEN": "ollama"
  }
}

# ~/.ywcoder.json
{
  "useOpenAIAdapter": true,
  "openAIModel": "llama2"
}
```

#### 内部 OpenAI 兼容网关

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://internal-api.company.com/v1",
    "ANTHROPIC_AUTH_TOKEN": "your-internal-api-key"
  }
}
```

### 禁用适配器

如果自动检测错误地启用了适配器，可以通过以下方式禁用：

```bash
export YWCODE_DISABLE_OPENAI_ADAPTER=1
```

或在 `~/.ywcoder.json` 中：

```json
{
  "useOpenAIAdapter": false
}
```

### 工作原理

当启用适配器时，YwCoder 会：
1. 启动一个本地 HTTP 代理服务器（端口 18000+）
2. 代理服务器接收 Anthropic 格式的请求
3. 将请求转换为 OpenAI 格式
4. 转发到实际的 OpenAI 兼容端点
5. 将响应转换回 Anthropic 格式
6. 返回给 Claude SDK

这个过程对 SDK 完全透明，SDK 仍然认为自己正在与 Anthropic API 通信。

### Q: 离线环境下 WebFetch 和 WebSearch 工具不可用？

A: 这是预期行为。离线配置已禁用这些工具。如需在部分联网环境中使用，请修改 `permissions.allow` 列表。

### Q: 如何创建多个配置方案（Profile）？

A: 在 YwCoder 设置页面点击 "Create Profile" 或手动创建：

```bash
# 创建开发环境配置
cp ~/.claude/settings.json ~/.claude/settings.development.json

# 创建生产环境配置
cp ~/.claude/settings.json ~/.claude/settings.production.json
```

然后在 VSCode 命令面板中运行 `YwCoder: Switch Profile` 切换。

## 安全建议

1. **API Key 管理**: 不要在版本控制中提交包含 API Key 的配置文件
2. **权限控制**: 根据实际需求调整 `permissions.allow` 和 `permissions.deny`
3. **沙盒启用**: 建议保持 `sandbox.enabled: true`
4. **定期清理**: 设置 `cleanupPeriodDays` 自动清理历史记录
