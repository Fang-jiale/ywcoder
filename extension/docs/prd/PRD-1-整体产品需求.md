## 1. 产品概述

### 1.1 产品名称
YW Coder（运维Coder）— 自研智能编程助手 VSCode 扩展

### 1.2 产品定位
深度集成于 VSCode 的智能编程助手，基于 Claude Code / Anthropic SDK 自研，为研发团队提供可控、可定制、可集成的 AI 编码体验。

### 1.3 目标用户
- 企业研发团队工程师
- 对代码安全有要求的内部开发者
- 需要深度定制 AI 工作流的开发团队

### 1.4 核心价值主张
| 维度 | 外部工具痛点 | YW Coder 解法 |
|------|------------|--------------|
| 数据安全 | 代码上传第三方云端 | 本地优先，可选私有部署 |
| 深度定制 | 功能封闭，无法扩展 | 自研架构，完全可控 |
| 系统集成 | 与内部系统割裂 | 可接入内部网关、MCP |
| 模型灵活 | 绑定单一供应商 | 多模型后端自由切换 |

---

## 2. 产品目标

### 2.1 业务目标
- 为团队提供安全可控的 AI 编码工具替代方案
- 沉淀 AI Agent 开发方法论与工程实践
- 形成可复用的模块化技术架构

### 2.2 技术目标
- 基于 VSCode Extension API 构建稳定可靠的扩展
- 实现前端（Vue 3）与扩展核心（TypeScript）的解耦通信
- 支持多种 AI 后端（Anthropic / OpenAI 兼容 / 本地模型）
- 流式响应，延迟 < 500ms 首字到达

### 2.3 用户目标
- 通过自然语言完成代码分析、生成、重构
- 在对话中直接执行文件操作和终端命令
- 管理多会话历史，支持上下文追溯
- 细粒度控制 AI 操作权限

## 4. 非功能需求

### 4.1 性能指标
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 扩展启动时间 | < 2s | 从 VSCode 激活到 WebView 可交互 |
| 首字延迟 | < 500ms | 用户发送消息到首字渲染 |
| 流式帧率 | > 15fps | 前端渲染流畅度 |
| 内存占用 | < 200MB | 扩展进程峰值内存 |

### 4.2 兼容性
| 环境 | 要求 |
|------|------|
| VSCode | ^1.98.0 |
| Node.js | >=18.0.0 |
| 操作系统 | Windows 10/11、macOS 10.15+、Linux |

### 4.3 安全要求
- 所有文件操作需用户显式授权
- 终端命令执行需审批（可配置自动批准模式）
- 网络访问（WebFetch/WebSearch）默认拒绝，需配置 allowlist
- 本地设置（settings.local.json）不提交到 git

### 4.4 可靠性
- 会话历史自动保存到 JSONL 格式文件
- 网络异常时支持断点续传
- AI 输出不确定时，测试聚焦"工具调用结构正确"和"消息渲染不崩溃"

## 5. 技术架构约束

### 5.1 分层原则
- **前端与核心解耦**：WebView 通过 BaseTransport 抽象与 Extension 通信，未来可替换为 WebSocket
- **核心与 AI 引擎解耦**：通过 Adapter 模式支持多模型后端
- **传输层与业务解耦**：ITransport 接口定义核心契约

### 5.2 依赖注入
- 使用自定义 DI 容器（InstantiationService + ServiceRegistry）
- 所有服务通过装饰器注入，支持懒加载
- 便于测试时替换为 Mock 实现

### 5.3 状态管理
- Extension 侧：无全局可变状态，通过服务接口交互
- WebView 侧：alien-signals（轻量级响应式）+ Pinia（结构化状态）

## 8. 术语表

| 术语 | 定义 |
|------|------|
| Agent | 具备自主决策、多轮迭代、自动执行能力的 AI 代理 |
| Tool Use | 大模型输出结构化工具调用请求的能力 |
| Content Block | 消息中的内容块，包括 text、tool_use、tool_result、thinking 等类型 |
| Profile | 配置方案，通过 `settings.<name>.json` 实现多配置切换 |
| MCP | Model Context Protocol，Anthropic 提出的模型上下文协议 |
| SSE | Server-Sent Events，服务端推送的流式传输协议 |
| JSONL | JSON Lines 格式，每行一个 JSON 对象，用于会话历史存储 |
